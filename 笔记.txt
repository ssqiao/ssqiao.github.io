编辑 index.md 或 index.html 以添加主页内容。

在本地启动 Jekyll 服务器，预览你的网站：
bundle exec jekyll serve
你可以通过 http://localhost:4000 查看效果。


克隆仓库到本地
打开终端或命令行工具，输入以下命令克隆仓库到本地：
git clone https://github.com/yourusername/yourusername.github.io.git

将更改提交到本地Git仓库并推送到GitHub：
git add .
git commit -m "Initial Jekyll site"
git push origin main



根目录内容：
README.md: 通常是项目的介绍和说明文档。
_pages/: 可能包含一些自定义页面的内容。# 暂时用不到
assets/: 包含网站使用的图片、PDF、bib、CSS、JavaScript等资源文件。
blog/: 可能是博客文章的存放目录。  # 可能用不到
feed.xml: RSS 订阅源文件。   # 如果你不打算让用户通过 RSS 订阅你的内容，那么可以选择不修改或删除这个文件。或者修改里面的boleizhou内容
index.html: 网站主页的主要 HTML 文件。
lab/: 可能是展示研究或实验室相关内容的页面。
news/: 存放news的页面。
others/: Awards和service栏目内容。
projects/: 存放项目展示的页面。
publications/: 发表文章或出版物的展示页面。
robots.txt: 告诉搜索引擎如何抓取网站的文件。
sitemap.xml: 网站地图，用于搜索引擎抓取。
talks/: 讲座或演讲的相关页面。
teaching/: 教学相关内容页面。

如果要将此代码改造成你的个人主页，以下文件和文件夹可能需要修改：
index.html: 修改主页内容，包括你的名字、简介、照片等。
_pages/: 这个文件夹可能包含一些自定义页面，例如关于我、简历等页面。你需要根据自己的需求修改这些页面。
assets/: 如果你要更改网站的样式或使用你自己的图片、PDF文件等，修改这个文件夹中的文件，如 CSS、JavaScript、图片等。
blog/, news/, projects/, publications/, lab/, teaching/, talks/: 这些文件夹对应网站的不同部分。你可以根据你的个人内容修改这些文件夹中的页面，例如添加或删除项目、博客文章、新闻、讲座等。
README.md: 如果你计划将该项目发布到 GitHub，建议修改这个文件，介绍你自己和你的网站。
sitemap.xml 和 robots.txt: 这些文件涉及 SEO 设置，可以根据需要进行调整。可以使用 Jekyll 的 jekyll-sitemap 插件自动生成 sitemap.xml 文件。这个插件会根据你的网站结构自动生成一个完整的站点地图，并在每次生成网站时自动更新。在 Gemfile 中添加：gem 'jekyll-sitemap';在 _config.yml 中添加：
plugins:
  - jekyll-sitemap。运行 bundle install 安装插件。每次运行 jekyll build 或 jekyll serve 时，插件会在生成的 _site 目录中自动创建或更新 sitemap.xml 文件。
feed.xml：尝试修改里面的boleizhou相关信息，替换成我的。或者jekyll-feed 是一个官方 Jekyll 插件，可以自动生成 feed.xml 文件。这个插件会根据你网站的内容自动创建一个标准的 RSS 或 Atom 订阅源。
